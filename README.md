# 德州撲克 AI 決策輔助系統

## 研究背景與動機

隨著線上博弈市場的蓬勃發展，德州撲克已然成為一項重要的線上遊戲。然而，玩家面臨著即時決策的巨大挑戰，需要在短時間內處理大量資訊，往往導致決策品質下降。面對這些挑戰，我們透過人工智慧技術來改變現有的遊戲格局，提升玩家在面對撲克賽局時的競爭力。

本研究旨在打造創新的德州撲克 AI 輔助系統，為玩家提供即時且最佳的策略建議，不僅能提升玩家的決策品質與遊戲樂趣，更可作為人工智慧技術與實際應用整合的突破性案例。

## 系統架構

本系統採用三層式架構，確保系統的模組化特性與可維護性：

### 前端展示層
- 採用現代化網頁技術建構使用者介面
- 使用 WebSocket 實現即時通訊功能
- 提供流暢的操作體驗與即時策略回饋

### 後端邏輯層
- 整合 Node.js 後端伺服器
- 配置 Python 運算環境
- 處理複雜的運算需求和即時通訊

### 資料儲存層
- 採用 MySQL 資料庫系統
- 實現資料的安全存取和有效管理
- 提供完整的資料備份機制

## 系統功能說明

### 影像辨識系統
影像辨識系統是本研究中最關鍵的技術元件之一，負責：
- 將螢幕上的牌局資訊轉換為可供分析的數位資料
- 採用 Python 開發的截圖模組結合 Anthropic Claude API
- 實現高精度的影像識別功能

### AI 決策系統
AI 決策系統採用深度學習技術，包含：
- 設計完整的三層全連接神經網路
- 包含輸入層、兩個隱藏層和輸出層
- 能夠有效處理複雜的牌局資訊
- 生成精確的策略建議

### 資料庫系統
資料庫系統採用關聯式資料庫 MySQL，包含：

使用者表（User）：
- 唯一識別碼（ID）
- 姓名（Name）
- 電子郵件（E-mail）
- 密碼（Password，使用 bcrypt 加密）
- 帳號建立時間（created_at）

遊戲歷史表（Game_History）：
- 遊戲階段（game_state）
- 玩家手牌（player_cards）
- 公共牌（board_cards）
- 底池大小（pot_size）
- 玩家位置（position）
- AI 決策建議（AI_decision）
- 玩家實際行動（Action_taken）

## 系統環境需求

### 硬體配置
- 處理器：Apple M 系列晶片（M1 或更新版本）
- 系統記憶體：16GB RAM 以上
- 儲存空間：256GB 以上
- 顯示器：支援 1920x1080 解析度以上

### 軟體環境
- macOS 最新版本
- Python 3.8+
- MySQL 8.0+
- Chrome 或 Safari 最新版本

## 系統操作流程

### 帳號管理
1. 帳號註冊流程
   - 進入系統註冊頁面
   - 填寫必要資訊
   - 系統驗證資訊完整性
   - 資料存入 MySQL 資料庫

2. 登入認證程序
   - 開啟系統登入頁面
   - 輸入註冊的電子郵件與密碼
   - 系統進行身份驗證
   - 驗證通過後進入主控台

### 主要功能操作

1. 系統啟動階段
   - 開啟線上德州撲克遊戲平台
   - 登入本輔助系統
   - 點擊「開始」按鈕啟動輔助功能

2. 即時操作階段
   - 按下空白鍵觸發影像擷取
   - 系統自動進行影像分析
   - 接收 AI 決策建議
   - 決策資訊自動儲存

3. 系統結束階段
   - 點擊「結束」按鈕
   - 確認資料保存完整
   - 可進行歷史記錄查看
